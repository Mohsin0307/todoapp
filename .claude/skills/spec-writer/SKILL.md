## When to Use
- Before /sp.specify
- When requirements are unclear
- When converting rough ideas to specs

## Procedure
1. Ask clarifying questions if needed
2. Write clear WHAT & WHY
3. Add constraints and assumptions
4. Output in /sp.specify-ready format

## Output Format
- Goal
- User Stories
- Constraints
- Non-goals
